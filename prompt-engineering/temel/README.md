# Prompt Engineering - Temel Notlar

Bu klasör, kendi prompt engineering prensiplerini ve kısa özet notlarını tutmak için.

## Kendi Kurallarım (taslak)
- Her zaman **bağlam** ver (dil, framework, dosya, versiyon).
- Modelden **rol** iste ("kıdemli X geliştirici" gibi).
- Çıktı formatını açık yaz (kod / madde madde / tablo vb.).
- Önce analiz, sonra çözüm, en sonra kod iste.
- Uzun işleri parçalara böl ("1) Analiz 2) Plan 3) Kod" gibi).
- İlk cevaptan sonra, eksikleri net yazarak ikinci tur iste.

Bu dosyayı kendi stiline göre düzenlemen için boş bıraktım; kuralları zamanla güncelleyebilirsin.
